# 双机保护手册

> 保护不是禁锢，是为了更自由地使用。
> 铁律限制的是破坏，不是创造。守护的目的是进化，不是停滞。

## desktop_guardian.ps1 — 使用指南

### 5种模式

| Action | 用途 |
|--------|------|
| `diagnose` | 23项安全检查（默认） |
| `fix` | 14项自动修复 |
| `protect` | 部署 agent_guard.json 安全边界 |
| `hosts-guard` | 持续监控 hosts（60s间隔） |
| `report` | JSON格式诊断报告 |

### 远程执行（从另一台电脑）

```powershell
# SSH
plink -batch -ssh user@target-ip -pw PASSWORD "powershell -NoProfile -ExecutionPolicy Bypass -File C:\path\desktop_guardian.ps1 -Action diagnose"

# WinRM
Invoke-Command -ComputerName target-ip -Credential $cred -ScriptBlock {
    & 'C:\path\desktop_guardian.ps1' -Action diagnose
}
```

## 铭文（每条背后都有真实的祸）

1. **不在443做转发** — portproxy抢windsurf-LG端口
2. **初始配置即最优** — Agent加proxy反而坏事
3. **账号必须有强密码** — 无密码账号=致命漏洞
4. **安全设置不可自动撤销** — unlock_all.bat每次开机瓦解安全
5. **先看日志再猜原因** — 多轮盲目诊断浪费时间
6. **Windows更新会回退设置** — RDP设置被重置
7. **IIS(W3SVC)不可Automatic** — HTTP.sys内核级锁443
8. **对照另一台机器找差异** — 比较版本/端口/配置
9. **勿停HTTP.sys** — WinRM依赖它，停了=断联
